Test

Write a test for the piece of functionality you are about to add.

Test from the user’s perspective first

Think about how the user will use it and write the test from that perspective. Don't create any classes or do any implementation work until you’ve done that.

Example

it "prints a statement with one deposit" do
  bank = Bank.new
  bank.deposit(1000)
  expect(bank.print_statement).to eq "1000"
end
